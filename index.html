Data Architecture
How data flows through the platform
Sales uploads are the engine. Everything — tickets, patient history, order availability — is derived from daily counter uploads. No complex integrations required.
Follow-Up Engine
How follow-up tickets are generated & prioritized
From 20k+ daily sales, the platform surfaces the 50–80 patients 3 CSRs can actually act on — using logic encoded from your admin's existing instincts.
Generation Flow
1
Counter uploads sales CSV
End of day. Contains: patient name, phone number, medicines bought, quantities, date.
2
Platform parses & maps to patient profiles
Phone number used as unique key. New patients created. Existing profiles updated with latest purchase.
3
Filter rules applied
System checks: repeat buyer? chronic medicine? cycle window approaching? Purchase value threshold?
4
Tickets generated & prioritized
Tier 1 first, then Tier 2. Daily cap configurable by Admin to avoid CSR overload.
5
CSR opens ticket board next morning
Ready list. No manual curation by admin required.
Prioritization Tiers
Tier 1
Chronic / Repeat Patients
Bought same medicine 2+ times. On BP, diabetes, thyroid, or other chronic meds. Refill cycle window approaching (e.g. bought 30-day supply 25+ days ago). Highest conversion probability. Follow up first.
Tier 2
Lapsed High-Value Patients
Previously regular buyer — no purchase in 30–45 days. Or single purchase above a set value threshold. Worth a recovery call. Assigned after Tier 1 tickets are covered.
Tier 3
Everyone Else
Not followed up manually in Phase 1. Future automation territory — SMS / WhatsApp reminders in Phase 3. System tracks them silently in the background.
⚙️ Admin Configurable Rules
Daily ticket cap per CSR · Cycle window thresholds · Medicine category tags (chronic vs acute) · Minimum purchase value for Tier 2

Online Order Flow
From follow-up call to fulfilled order

Counter User
Counter dashboard: uploads & orders
Designed to fit into an existing daily routine with minimal friction. Two daily tasks + order management.
Daily Upload Flow
1
Export from existing billing/ERP system
Counter exports sales & stock as CSV or Excel — same format they use today. No change to their process.
2
Upload Sales CSV to platform
Simple drag & drop or file picker. Platform confirms row count and flags obvious errors.
3
Upload Stock CSV to platform
Current stock snapshot. Used by CSR to verify availability before placing orders.
4
Dashboard confirms upload status
Green checkmark = today's data is live. Missed upload? Platform flags it for admin visibility.
⚠️ Important Design Decision
Sales and stock data are not cross-validated. Uploaded file = source of truth. No reconciliation required. Keeps it simple and scalable.

Technical Design Decision
Handling inconsistent item data across counters
Counters use different item names and codes. We don't unify everything — we use a confidence-based partial mapping that's practical and maintainable.

Status
Stored as-is. Unmapped.
Unmapped items still appear in patient history as purchased — they just don't link to a master item. CSR can still follow up. Admin can resolve mappings over time.
Our Strategy
🏷️
Fuzzy Name Matching
String similarity matching between counter item names and master list. High-confidence matches auto-linked. Medium flagged for admin review.
✋
Manual Override UI
Admin can manually link any counter item to a master item. One-time action. All future uploads auto-map using that link.
💊
Medicine Tags for Follow-Up
CSR can tag any medicine — mapped or unmapped — as "chronic" to include in follow-up logic. Mapping status doesn't block follow-up creation.
📈
Improves Over Time
As admin resolves mappings, coverage improves. Starts useful on day one — gets smarter each week without any system overhaul.
Delivery Roadmap
Phased build. Prove it, then scale.
Phase 1 is self-contained and delivers measurable value. Phase 2 and 3 activate only after Phase 1 is validated.

The foundation. Everything that replaces the Excel + WhatsApp workflow today.
Counter Upload Dashboard

Patient Profile System
Auto-created from sales uploads. Phone as unique key. Cross-counter history.
Follow-Up Ticket Engine
Tier 1 & 2 generation rules. Daily cap. CSR assignment.
CSR Ticket Board
Prioritized daily list. Call outcome logging. Snooze & reschedule.
Online Order Flow
CSR creates order → counter notified → accept/decline → follow-up scheduled.
Admin Dashboard
CSR performance, upload compliance, basic sales visibility.
Item Mapping (Partial)
Fuzzy match + manual override. Chronic medicine tagging.
→ Pilot: 2 counters · Target: 60 days
Phase 2 — Post Validation
B2C & Order Intelligence
Expand when Phase 1 follow-up conversion is proven and stable.
Patient Self-Ordering Portal
Customers browse, request medicines directly. CSR reviews & confirms.
Real-Time Stock Engine
Live availability check across counters before order confirmation.
CSR Admin Role (CSR Admin)
Senior CSR can manage junior CSRs, reassign tickets, view team performance.
Enhanced Item Mapping
Bulk mapping tools. Mapping confidence reports for admin.
Order History & Billing Ref
Full order trail per patient. Counter billing reference tracking.
Trigger: Phase 1 stable for 60+ days
Phase 3 — Scale
Automation & Intelligence
Automate what Phase 1 & 2 proved works manually.
WhatsApp / SMS Reminders
Auto-triggered refill reminders at cycle day 25. Replaces Tier 3 manual follow-up.
Dedicated Mobile App
Push notifications for patients. In-app ordering. Prescription history.
Retention Analytics
Cohort tracking. Churn prediction. Revenue impact per CSR action.
Multi-Counter Routing
Auto-assign orders to nearest/best-stocked counter. Reduce manual CSR intervention.
Demand Forecasting
Predict stock needs per counter based on patient cycle data patterns.
Trigger: Phase 2 stable + volume growth
